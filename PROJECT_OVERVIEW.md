# AI PPT Maker — Project Overview

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 15 (App Router, Turbopack) |
| Language | TypeScript 5 (strict) |
| UI | React 18, Tailwind CSS v4, shadcn/ui (Radix) |
| Auth | NextAuth v4 + Google OAuth + Prisma Adapter |
| AI | Google Gemini 1.5 Pro (text) + Imagen 3 (images) |
| Database | PostgreSQL on NeonDB via Prisma ORM |
| Payments | LemonSqueezy (subscription + template marketplace) |
| Upload | UploadCare |
| State | Zustand stores |

## Request Lifecycle

1. Browser requests a protected route (e.g., `/dashboard`)
2. `middleware.ts` checks JWT via NextAuth — redirects to `/sign-in` if unauthenticated
3. Server Component loads, calls `onAuthenticateUser()` to verify session + fetch prisma User
4. Data is fetched in the Server Component (e.g., `getRecentProjects()`)
5. Client components receive data as props; Zustand stores hold ephemeral UI state

## PPT Generation Flow

```
User types prompt in CreativeAI component
    ↓
generateCreativePrompt(prompt) [Server Action]
    → Gemini 1.5 Pro generates 5+ outline points as JSON
    ↓
createProject(title, outlines) [Server Action]
    → Saves project to DB with outlines array
    → Redirects to /presentation/[id]/select-theme
    ↓
User picks a theme
    ↓
generateLayout(projectId, theme) [Server Action]
    → Checks auth + subscription (bypassed with DEV_BYPASS_PAYMENTS=true)
    → generateLayoutsJSON(outlines)
        → Gemini 1.5 Pro generates Slide JSON (layout types + content types)
        → For each image slot: calls Imagen 3 API → returns base64
    → Saves slides JSON to project in DB
    → Redirects to /presentation/[id] (the editor)
    ↓
User edits slides in the drag-and-drop editor
    ↓
updateSlides(projectId, slides) [Server Action] — saves on change
```

## Database Schema

```
User
  id (cuid) ← primary key, created by NextAuth PrismaAdapter
  email (unique)
  name, image, emailVerified ← populated by NextAuth from Google
  subscription: Boolean ← set to true when payment webhook fires
  lemonSqueezyAPIKey, storeID, webhookSecret ← for template reselling
  
Account ← NextAuth OAuth tokens (provider: "google")
  userId → User.id

Project
  id (cuid)
  userId → User.id
  title, outlines: String[]
  slides: Json? ← the full Slide[] array (generated by AI)
  themeName, thumbnail
  isDeleted: Boolean ← soft delete
  isSellable: Boolean ← for template marketplace
  variantId ← LemonSqueezy variant for template sales
```

## Slide Data Structure

Slides are stored as JSON in `Project.slides`. Structure:

```typescript
type Slide = {
  id: string;
  slideName: string;
  type: string;          // Layout type: "accentLeft", "twoColumns", etc.
  className: string;
  content: ContentItem;  // Recursive tree of content elements
}

type ContentItem = {
  id: string;
  type: ContentType;     // "column", "heading1", "paragraph", "image", etc.
  name: string;
  content: string | ContentItem | ContentItem[];
  className?: string;
  placeholder?: string;
  alt?: string;
}
```

## Authentication Flow (NextAuth + Google)

1. User visits `/sign-in` → clicks "Continue with Google"
2. `signIn("google", { callbackUrl: "/dashboard" })` → redirects to Google
3. Google redirects to `/api/auth/callback/google` (NextAuth handles)
4. **PrismaAdapter automatically creates** `User` + `Account` records in DB
5. JWT is created with `user.id` as `token.id`
6. On each request, `getServerSession(authOptions)` returns `{ user: { id, email, name, image } }`

## Adding Future Features

- **New slide layout type**: Add to `lib/slideLayouts.ts`, register in `lib/constant.ts`, add to the AI prompt in `actions/ai.ts`
- **New content type**: Add to `ContentType` union in `lib/types.ts`, render in `editor/masterRecursiveComponent.tsx`
- **New AI provider**: Create a new file in `src/actions/`, swap in `genAI` equivalent
- **Export to PPTX**: Add a server action that maps Slide JSON → pptxgenjs format
